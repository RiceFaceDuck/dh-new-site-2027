# Backoffice Development Guidelines

เมื่อมีการพัฒนาหรือเพิ่มฟีเจอร์ใหม่ใน **ระบบหลังบ้าน (dh-backoffice-react)** ห้ามลืมทำสิ่งเหล่านี้เด็ดขาด:
1. **ต้องเพิ่มคำอธิบายและการสอนการใช้งานเข้าไปใน UI ของระบบด้วยทุกครั้ง** (In-App Documentation)
2. คู่มือ/คำอธิบายต้องประกอบไปด้วย:
   - **ตำรา / คำอธิบาย**: อธิบายว่าฟีเจอร์นี้คืออะไร ใช้ทำอะไร
   - **วิธีการใช้งาน (How-to)**: อธิบายเป็นขั้นเป็นตอน
   - **เทคนิคการใช้งาน (Tips & Tricks)**: บอกเทคนิคที่ทำให้ใช้งานได้ง่ายหรือเร็วขึ้น (เช่น Shortcut Keys)
   - **ตัวอย่างผลลัพธ์ (Expected Results)**: แสดงตัวอย่างหรือคำเตือนที่ชัดเจนก่อนผู้ใช้จะกดยืนยันการทำงาน

ให้คำนึงเสมอว่าผู้ใช้งานมีหลายระดับ (Staff ใหม่, Manager) การสร้าง UI ต้องมาพร้อมคำอธิบายในตัวเสมอ (เช่น Guide Modal, Tooltip, Help Panel)
